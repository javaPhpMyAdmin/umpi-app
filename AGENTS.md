# Umpi App — Agent Guide

## Stack
- Expo SDK 54 + React Native 0.81 + TypeScript 5.9 + React 19.1
- File-based routing via **Expo Router** (entry: `expo-router/entry` in `package.json#main`)
- **Supabase** auth + database (client in `lib/supabase.ts`)
- Path alias `@/*` → project root (configured in `tsconfig.json`)

## Essential commands
```bash
npm run dev          # start Expo dev server
npm run build:web    # build for web (expo export --platform web)
npm run lint         # expo lint
npm run typecheck    # tsc --noEmit (strict mode)
npm run format       # (npx prettier --write .)
```

No test framework is configured.

## Environment
- `EXPO_PUBLIC_SUPABASE_URL` and `EXPO_PUBLIC_SUPABASE_ANON_KEY` are required at runtime.
- `.env` is **not** gitignored (contains Supabase keys). Do not commit if values are sensitive.
- `expo-env.d.ts` is gitignored (auto-generated by Expo).

## Architecture
- **`app/`** — Expo Router routes (Spanish-language classifieds marketplace)
  - `(tabs)/` — main tab navigator: Inicio, Explorar, Publicar, Mensajes, Perfil
  - Standalone: `login.tsx`, `register.tsx`, `plans.tsx`, `settings.tsx`
  - Groups: `admin/`, `chat/`, `listing/`
- **`contexts/AuthContext.tsx`** — wraps app root; provides Supabase auth + profile CRUD
- **`lib/supabase.ts`** — Supabase client init with custom storage adapter
- **`lib/storage.ts`** — cross-platform storage (localStorage on web, Map fallback on native)
- **`types/index.ts`** — Listing, Profile, Category, Conversation, Message, SubscriptionPlan
- **`constants/mockData.ts`** — fallback seed data used when Supabase calls return null
- **`supabase/migrations/`** — SQL schema + seed data for Supabase

## Conventions
- **Prettier**: singleQuote, tabWidth 2, no tabs (`.prettierrc`)
- **Components**: default exports, functional, `StyleSheet.create` at bottom of file
- **Colors**: defined in `constants/colors.ts` (Spanish-named palette)
- **Mock data fallback**: screens fetch from Supabase but fall back to `mockData` imports if data is null

## Gotchas
- `expo-env.d.ts` is generated at build time and gitignored — typecheck might fail until `npm run dev` has run once.
- `nativewind-env.d.ts` is referenced in `tsconfig.json` includes but no NativeWind dependency exists — ignore typecheck noise about this file.
- All UI text is in Spanish (Argentina locale). Prices use `es-AR` formatting.
